# GoPrompt: Professional Teleprompter App

GoPrompt is a professional-grade iOS teleprompter application completely redesigned with a premium, glassmorphic aesthetic (derived from Stitch) and governed by deep, hardware-accelerated camera features.

## 🌟 Features
- **Premium Glassmorphic UI**: High-contrast dark mode aesthetics with translucent panels, rounded pill toolbars, and clear SF Pro typography.
- **Advanced Camera Controls**: Full manual control over ISO, Exposure, White Balance, and Focus using custom sliders.
- **Cinematic Depth of Field**: Software-simulated bokeh and aperture control via `AVSemanticSegmentationMatte`.
- **Green Screen (Chroma Key)**: Real-time background replacement using Metal and CoreImage shaders.
- **Distraction-Free Teleprompter**: Buttery-smooth text scrolling based on configurable Words-Per-Minute (WPM), featuring mirror text, adjustable typography, and smart pause-on-tap functionality.
- **Script Management**: Create, edit, and organize speeches with auto-calculated read times and word counts.
- **Hardware Integrations**: Support for external displays, auto-framing, and cinematic video stabilization.

## 🏗️ Architecture & Frameworks

### 1. AVFoundation Pipeline
The app features a low-latency custom `AVCaptureSession` pipeline. Rather than using the native `AVCaptureVideoPreviewLayer`, the app intercepts standard and depth-mapped video buffers via `AVCaptureVideoDataOutput`, piping them explicitly through a `CIContext` for real-time visual manipulation.

### 2. Metal & CoreImage Integration (MTKView)
To support real-time Chroma Keying and Cinematic Blur with zero dropped frames, the app uses a custom `MetalPreviewView` backed by an `MTKView`. 
- **Chroma Key**: Custom CoreImage filters analyze the chroma spectrum to isolate green values, subsequently replacing them with a custom background graphic.
- **Depth Blur**: Evaluates the subject-matte generated by the neural engine (`AVSemanticSegmentationMatte`) and applies a `CIMaskedVariableBlur` to accurately simulate DSLR optical bokeh.

### 3. Physical Rotation Architecture
To handle device orientation flawlessly, GoPrompt shifts rotation responsibility from the layout engine to the views. The `CameraPreviewView` and floating overlays apply direct `CGAffineTransform` rotation constraints responding to hardware sensors. This **Animation Isolation Pattern** completely eliminates the catastrophic SwiftUI layout engine jitter that often plagues teleprompters during mid-recording orientation changes.

### 4. SwiftUI & Design System
The user interface is 100% native SwiftUI, unified by a central `DesignSystem.swift`.
- Extensive use of `.ultraThinMaterial` and nested `ZStack` elements achieve the premium glassmorphic UX heavily prominent in Apple's VisionOS paradigms.
- State is strictly managed through robust `ObservableObjects` such as `CinematicCameraService` mitigating UI synchronization bugs.

## 📸 Stitch Screen Designs

*The application interface has been fully modeled upon the following generated UX artifacts:*

- **Recording View** (`RecordingView.swift`): Full-screen camera feed with highly translucent bottom pill controls and a floating teleprompter overlay.
- **Camera Configuration** (`CameraControlsOverlay.swift`): Retractable frosted-glass modal arrays granting granular access to ISO/Exposure/Apperture.
- **Script Library** (`ScriptListView.swift`): Grid-based cards highlighted by high-contrast primary accents.
- **Script Editor** (`ScriptEditorView.swift`): Distraction-free, fully-darkened text composition framework.
- **Settings** (`SettingsView.swift`): A custom-built scrollable layout intentionally diverging from standard iOS `Form` styles to preserve UI integrity.
- **Recording Gallery** (`RecordingGalleryView.swift`): A localized springboard directly launching the user into their native high-quality Photos Library.
